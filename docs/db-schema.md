## Entity Relationship Overview

```
users
  │
  ├──< conversations (user_id → users.id)
  │         │
  │         └──< messages (conversation_id → conversations.id)
  │
  └──< tickets (user_id → users.id)
            │
            └── conversation_id → conversations.id (nullable)
```

**Cardinality:**
- One `user` has many `conversations`
- One `conversation` has many `messages`
- One `user` has many `tickets`
- One `ticket` optionally links to one `conversation`
- One `conversation` optionally links to one `ticket` (via tickets.conversation_id)

## Table Definitions

### users

Stores all registered users across all roles.

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | Primary key |
| `name` | VARCHAR(100) | No | — | Full display name |
| `email` | VARCHAR(255) | No | — | Unique login email |
| `password_hash` | VARCHAR(255) | No | — | bcrypt hash of password |
| `role` | ENUM(UserRole) | No | `customer` | Access role |
| `is_active` | BOOLEAN | No | `true` | Account status flag |
| `created_at` | TIMESTAMPTZ | No | `NOW()` | Account creation timestamp |
| `updated_at` | TIMESTAMPTZ | No | `NOW()` | Last update timestamp |

**Constraints:**
- `id` — PRIMARY KEY
- `email` — UNIQUE
- `role` — CHECK in (`customer`, `admin`)

---

### conversations

Represents a support conversation thread across any channel.

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | Primary key |
| `user_id` | UUID | No | — | FK → users.id |
| `channel` | ENUM(ConversationChannel) | No | `web` | Originating channel |
| `status` | ENUM(ConversationStatus) | No | `active` | Current state |
| `subject` | VARCHAR(200) | Yes | `NULL` | Conversation subject line |
| `created_at` | TIMESTAMPTZ | No | `NOW()` | Creation timestamp |
| `updated_at` | TIMESTAMPTZ | No | `NOW()` | Last update timestamp |

**Constraints:**
- `id` — PRIMARY KEY
- `user_id` — FOREIGN KEY → `users(id)` ON DELETE CASCADE

---

### messages

Individual messages within a conversation. Stores both customer and AI messages.

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | Primary key |
| `conversation_id` | UUID | No | — | FK → conversations.id |
| `sender_type` | ENUM(SenderType) | No | — | Who sent this message |
| `content` | TEXT | No | — | Raw message text |
| `intent` | VARCHAR(50) | Yes | `NULL` | AI-classified intent label |
| `ai_confidence` | FLOAT | Yes | `NULL` | AI classification confidence (0.0–1.0) |
| `metadata` | JSONB | Yes | `{}` | Extensible metadata (model info, escalation flags) |
| `created_at` | TIMESTAMPTZ | No | `NOW()` | Message timestamp |

**Constraints:**
- `id` — PRIMARY KEY
- `conversation_id` — FOREIGN KEY → `conversations(id)` ON DELETE CASCADE
- `ai_confidence` — CHECK (`ai_confidence >= 0.0 AND ai_confidence <= 1.0`)

**Notes on `metadata` (JSONB):**
For AI-generated messages, `metadata` typically contains:
```json
{
  "model": "gpt-4o-mini",
  "escalated": false,
  "escalation_reason": null,
  "prompt_tokens": 312,
  "completion_tokens": 89
}
```

### tickets

Support tickets linked to users and optionally to conversations.

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `id` | UUID | No | `gen_random_uuid()` | Primary key |
| `user_id` | UUID | No | — | FK → users.id |
| `conversation_id` | UUID | Yes | `NULL` | FK → conversations.id (optional) |
| `title` | VARCHAR(200) | No | — | Short ticket title |
| `description` | TEXT | No | — | Full issue description |
| `category` | ENUM(TicketCategory) | Yes | `general` | Issue category |
| `priority` | ENUM(TicketPriority) | No | `medium` | Ticket urgency |
| `status` | ENUM(TicketStatus) | No | `open` | Current lifecycle state |
| `assigned_to` | UUID | Yes | `NULL` | FK → users.id (admin assignee) |
| `created_at` | TIMESTAMPTZ | No | `NOW()` | Creation timestamp |
| `updated_at` | TIMESTAMPTZ | No | `NOW()` | Last update timestamp |

**Constraints:**
- `id` — PRIMARY KEY
- `user_id` — FOREIGN KEY → `users(id)` ON DELETE CASCADE
- `conversation_id` — FOREIGN KEY → `conversations(id)` ON DELETE SET NULL
- `assigned_to` — FOREIGN KEY → `users(id)` ON DELETE SET NULL

## Foreign Key Relationships

```
users.id ◄─── conversations.user_id           (CASCADE DELETE)
users.id ◄─── tickets.user_id                 (CASCADE DELETE)
users.id ◄─── tickets.assigned_to             (SET NULL)
conversations.id ◄─── messages.conversation_id (CASCADE DELETE)
conversations.id ◄─── tickets.conversation_id  (SET NULL)
```

**Cascade Behavior:**
- Deleting a `user` cascades to all their `conversations` and `tickets`.
- Deleting a `conversation` cascades to all its `messages`. Linked tickets have `conversation_id` set to NULL.
- Deleting a message does not affect anything else.

## SQL Reference

These statements are provided as reference documentation. Schema is managed by Alembic migrations — do not run these manually in production.

```sql
-- Enable UUID generation
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

-- Enum types
CREATE TYPE user_role AS ENUM ('customer', 'admin');
CREATE TYPE conversation_status AS ENUM ('active', 'resolved', 'escalated', 'closed');
CREATE TYPE conversation_channel AS ENUM ('web', 'web_form', 'email', 'whatsapp');
CREATE TYPE sender_type AS ENUM ('customer', 'ai', 'agent', 'system');
CREATE TYPE ticket_status AS ENUM ('open', 'in_progress', 'resolved', 'closed');
CREATE TYPE ticket_priority AS ENUM ('low', 'medium', 'high', 'urgent');
CREATE TYPE ticket_category AS ENUM ('billing', 'technical', 'account', 'shipping', 'general');

-- Users table
CREATE TABLE users (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name        VARCHAR(100) NOT NULL,
    email       VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    role        user_role NOT NULL DEFAULT 'customer',
    is_active   BOOLEAN NOT NULL DEFAULT TRUE,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_users_role ON users(role);

-- Conversations table
CREATE TABLE conversations (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    channel     conversation_channel NOT NULL DEFAULT 'web',
    status      conversation_status NOT NULL DEFAULT 'active',
    subject     VARCHAR(200),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_conversations_user_id ON conversations(user_id);
CREATE INDEX idx_conversations_status ON conversations(status);
CREATE INDEX idx_conversations_channel ON conversations(channel);
CREATE INDEX idx_conversations_created_at ON conversations(created_at DESC);

-- Messages table
CREATE TABLE messages (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    conversation_id UUID NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
    sender_type     sender_type NOT NULL,
    content         TEXT NOT NULL,
    intent          VARCHAR(50),
    ai_confidence   FLOAT CHECK (ai_confidence >= 0.0 AND ai_confidence <= 1.0),
    metadata        JSONB DEFAULT '{}',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_messages_conversation_id ON messages(conversation_id);
CREATE INDEX idx_messages_sender_type ON messages(sender_type);
CREATE INDEX idx_messages_created_at ON messages(created_at);

-- Tickets table
CREATE TABLE tickets (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    conversation_id UUID REFERENCES conversations(id) ON DELETE SET NULL,
    title           VARCHAR(200) NOT NULL,
    description     TEXT NOT NULL,
    category        ticket_category DEFAULT 'general',
    priority        ticket_priority NOT NULL DEFAULT 'medium',
    status          ticket_status NOT NULL DEFAULT 'open',
    assigned_to     UUID REFERENCES users(id) ON DELETE SET NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_tickets_user_id ON tickets(user_id);
CREATE INDEX idx_tickets_status ON tickets(status);
CREATE INDEX idx_tickets_priority ON tickets(priority);
CREATE INDEX idx_tickets_assigned_to ON tickets(assigned_to);
CREATE INDEX idx_tickets_conversation_id ON tickets(conversation_id);
CREATE INDEX idx_tickets_created_at ON tickets(created_at DESC);

-- Auto-update updated_at timestamps
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_users_updated_at
    BEFORE UPDATE ON users
    FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER trg_conversations_updated_at
    BEFORE UPDATE ON conversations
    FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER trg_tickets_updated_at
    BEFORE UPDATE ON tickets
    FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

## Data Integrity Notes

1. **Password Storage** — Passwords are never stored in plaintext. Only bcrypt hashes (cost factor 12) are persisted in `password_hash`.

2. **Soft Deletes** — Users are never hard-deleted in normal operation; `is_active = false` is used to deactivate accounts. Hard cascade deletes exist as a last resort.

3. **AI Confidence** — `ai_confidence` is `NULL` for non-AI messages. Application code should check `sender_type == 'ai'` before reading this field.

4. **JSONB Metadata** — The `metadata` column on `messages` is schema-free. Consumers should handle missing keys gracefully. The `metadata` field is indexed with a GIN index if frequent JSON key lookups are needed at scale.

5. **UUID Generation** — All primary keys use UUID v4 (`gen_random_uuid()`). This avoids sequential ID enumeration and supports distributed insert scenarios.

6. **Timezone Handling** — All timestamps use `TIMESTAMPTZ` (timestamp with time zone). All values are stored in UTC. Client applications are responsible for local timezone conversion.
